# EXE 파일 수익화 전략 가이드

## 🎯 수익화 방법 개요

Electron 기반 EXE 파일로 수익화하는 주요 방법들을 정리했습니다.

---

## 1. 라이선스 키 시스템 (License Key System) ⭐ 추천

### 작동 방식
- 사용자가 구매한 라이선스 키를 입력하여 앱 활성화
- 온라인 인증 서버(Vercel + MongoDB)와 연동하여 키 검증
- 키는 일회성 또는 기간 제한 가능

### 장점
- ✅ 가장 일반적이고 검증된 방법
- ✅ 사용자별 관리 가능
- ✅ 키 재사용 방지 가능
- ✅ 기간 제한 설정 가능 (1개월, 1년 등)

### 구현 필요 사항
1. **백엔드 API (Vercel)**
   - 라이선스 키 생성 엔드포인트
   - 라이선스 키 검증 엔드포인트
   - 사용 횟수/기간 추적

2. **MongoDB 스키마**
   ```javascript
   {
     licenseKey: String (unique),
     userId: String,
     createdAt: Date,
     expiresAt: Date,
     isActive: Boolean,
     activationCount: Number,
     maxActivations: Number,
     hardwareId: String[] // 기기별 활성화 추적
   }
   ```

3. **Electron 앱 수정**
   - 라이선스 키 입력 UI 추가
   - 앱 시작 시 라이선스 검증
   - 로컬에 암호화된 라이선스 정보 저장

---

## 2. 프리미엄 기능 잠금 (Freemium Model)

### 작동 방식
- 기본 기능은 무료 제공
- 고급 기능(고해상도, 배치 변환 등)은 유료 잠금

### 장점
- ✅ 사용자가 먼저 체험 가능
- ✅ 전환율 높음
- ✅ 무료 사용자도 앱 홍보 효과

### 구현 예시
```javascript
// 무료: 720p까지만
// 프리미엄: 1080p, 4K, 배치 변환, 워터마크 제거 등
```

---

## 3. 사용 횟수 제한 (Usage Limits)

### 작동 방식
- 무료: 일일/월간 변환 횟수 제한 (예: 5회/일)
- 유료: 무제한 또는 높은 제한

### 장점
- ✅ 간단한 구현
- ✅ 사용자가 필요성 느끼기 쉬움

---

## 4. 구독 모델 (Subscription Model)

### 작동 방식
- 월간/연간 구독으로 앱 사용
- 정기적으로 서버에서 구독 상태 확인

### 장점
- ✅ 지속적인 수익
- ✅ 업데이트 제공 명확

---

## 5. 일회성 구매 (One-time Purchase)

### 작동 방식
- 한 번 구매하면 평생 사용
- 라이선스 키로 활성화

### 장점
- ✅ 사용자에게 부담 적음
- ✅ 높은 전환율 가능

---

## 🛠️ 추천 구현: 하이브리드 모델

**무료 버전 + 라이선스 키 시스템** 조합을 추천합니다:

1. **무료 버전**
   - 기본 변환 기능 제공
   - 720p 해상도까지만
   - 일일 3회 제한

2. **프리미엄 버전 (라이선스 키)**
   - 모든 해상도 (1080p, 4K)
   - 무제한 변환
   - 배치 변환
   - 우선 지원

---

## 📋 구현 단계

### Phase 1: 백엔드 구축 (Vercel + MongoDB)

1. **API 엔드포인트 생성**
   ```
   POST /api/license/verify
   POST /api/license/generate
   GET /api/license/status
   ```

2. **MongoDB 모델 생성**
   - 라이선스 키 관리
   - 사용자 활성화 추적

### Phase 2: Electron 앱 수정

1. **라이선스 관리 모듈 추가**
   - 키 입력 UI
   - 서버 검증 로직
   - 로컬 저장 (암호화)

2. **기능 제한 로직 추가**
   - 무료/프리미엄 분기
   - 사용 횟수 추적

### Phase 3: 결제 시스템 연동

1. **결제 게이트웨이 선택**
   - Stripe (국제)
   - 토스페이먼츠 (국내)
   - PayPal

2. **구매 플로우**
   - 웹사이트에서 결제
   - 라이선스 키 자동 발급
   - 이메일로 키 전송

## 🔒 보안 고려사항

1. **라이선스 키 암호화**
   - 로컬 저장 시 암호화
   - 네트워크 전송 시 HTTPS

2. **하드웨어 ID 바인딩**
   - 기기별 활성화 제한 (예: 3대까지)
   - 재사용 방지

3. **서버 검증**
   - 클라이언트만의 검증은 우회 가능
   - 반드시 서버에서 검증

4. **오프라인 모드**
   - 일정 기간 오프라인 사용 허용
   - 주기적으로 온라인 검증
